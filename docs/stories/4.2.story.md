# Story 4.2: Item Quantity Selection

## Status: Done

## Story

**As a** user,
**I want** to set the quantity of each item I include in my subkit,
**so that** I can account for my household size and specific needs.

## Acceptance Criteria

1. Each included item displays a quantity selector with increment and decrement buttons
2. Quantity selector is only active when an item is toggled as included
3. Minimum quantity is 1 when an item is included
4. Maximum quantity per item is 10 for MVP
5. Quantity defaults to 1 when an item is first included
6. Quantity changes are reflected immediately in application state
7. Item quantity is displayed on the Summary Page

## Dev Notes

### Critical Layout Rule
[Source: docs/architecture/11-frontend-developer-standards.md, Rule 8]
- QuantitySelector container ALWAYS reserves its layout space
- Only `opacity` and `pointer-events` change
- Never conditionally render or hide with `display:none`
- This prevents CLS (Cumulative Layout Shift) — UX spec Section 10

### Store Action
[Source: docs/architecture/5-state-management.md#store]
- `setItemQuantity(subkitId, itemId, qty)`: clamps to `Math.max(1, Math.min(10, qty))`
- No-op if item not in `itemSelections`

### UX Spec Flow 3
[Source: UX Spec Section 3, Flow 3]
- Toggle ON → included state; qty selector activates at 1; max 10
- Toggle OFF → excluded; qty resets to 1 internally
- `−` at qty 1 → disabled; does not toggle item off
