# Story 12.1: Item Price Display on ItemCard

## Status: Ready for Review

## Story

**As a** kit builder user,
**I want** to see the per-unit price of each item displayed on its card while configuring my subkits,
**so that** I have continuous cost visibility as I build my kit without needing to open the cart sidebar.

## Acceptance Criteria

1. **AC1 — Price renders below star rating when `pricePlaceholder` is non-null.**
   In `src/components/item-config/ItemCard.tsx`, a new price element renders inside the toggle button area, below the `StarRating` component (or below the description `<p>` if `item.rating` is `null`). The price text reads `$XX.XX` — computed as `'$' + item.pricePlaceholder.toFixed(2)`.

2. **AC2 — Price typography and color match spec.**
   The price element uses `text-xs` (12px) with `font-normal` (400 weight) and color `text-[var(--color-neutral-500)]` — matching the `text-caption` / `neutral-500` spec from the front-end spec.

3. **AC3 — No price element rendered when `pricePlaceholder` is `null`.**
   When `item.pricePlaceholder` is `null`, no price element, no placeholder text, and no empty container is rendered. The layout collapses as if the price element does not exist.

4. **AC4 — No changes to `ItemCardProps` interface.**
   The price is read directly from the existing `item.pricePlaceholder` field on the `KitItem` type already available via the `item` prop. No new prop is added to `ItemCardProps`.

5. **AC5 — Price visible on both `ItemConfigScreen` and `CustomSubkitScreen`.**
   Since both screens render `ItemCard` with the same `item` prop containing `pricePlaceholder`, the price display appears on both standard subkit item configuration and the custom subkit cross-category browser with no additional wiring.

6. **AC6 — Existing ItemCard tests continue to pass.**
   All 11 existing tests in `tests/components/ItemCard.test.tsx` pass without modification. The existing mock item has `pricePlaceholder: null`, so no price renders in those tests — they are unaffected.

7. **AC7 — New ItemCard tests added to `tests/components/ItemCard.test.tsx`.**

   | Test | Assertion |
   |---|---|
   | Price renders for item with non-null pricePlaceholder | `screen.getByText('$29.99')` present (mock item with `pricePlaceholder: 29.99`) |
   | Price does not render for item with null pricePlaceholder | `screen.queryByText(/\$/)` returns null (existing mock item with `pricePlaceholder: null`) |
   | Price renders below star rating when rating exists | Price element exists in DOM after StarRating (mock item with both `rating` and `pricePlaceholder` set) |
   | axe accessibility passes with price displayed | `expect(results).toHaveNoViolations()` on card with price |

8. **AC8 — TypeScript strict compilation and all tests pass.**
   `npm run typecheck` and `npm run test:run` both pass with zero errors.

### Current ItemCard Component Structure
[Source: src/components/item-config/ItemCard.tsx]

The `ItemCard` component currently renders (in order within the toggle button area):
1. `ImageWithFallback` — item image or category fallback
2. A `<div className="flex flex-col gap-1 p-3">` containing:
   - `<span>` with item name (`text-sm font-semibold`)
   - `<p>` with item description (`text-xs text-neutral-500`)
   - Conditional `StarRating` (when `item.rating != null && item.reviewCount != null`)

The price element should be added as a new conditional **after** the `StarRating` block, inside the same `<div>`. This places it below the star rating when present, or below the description when no rating exists.

The `item` prop is typed as `KitItem`, which already includes `pricePlaceholder: number | null` — no prop interface change needed.
