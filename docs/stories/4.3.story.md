# Story 4.3: Empty Subkit Option

## Status: Done

## Story

**As a** user,
**I want** the option to order a completely empty subkit container,
**so that** I can use the storage slot for supplies I already own without purchasing duplicate items.

## Acceptance Criteria

1. Each subkit configuration screen includes a clearly labeled empty container option
2. Selecting empty container deselects all items and disables the item list for that subkit
3. A clear visual indicator confirms the subkit is set to empty
4. The user can reverse this selection at any time and re-enable item selection
5. The Summary Page clearly indicates which subkits are empty containers
6. Empty container subkits still count toward the 3-6 subkit constraint

## Dev Notes

### Architecture Rule 9
[Source: docs/architecture/11-frontend-developer-standards.md, Rule 9]
- EmptyContainerOption is rendered on BOTH `ItemConfigScreen` AND `CustomSubkitScreen`
- Custom subkit supports empty container option identically to standard subkits per PRD FR9
- Behavior: deselects all Custom items, dims item grid, displays inline confirmation in Custom category color (#3730A3), reflected on Summary Page

### Store Action
[Source: docs/architecture/5-state-management.md#store]
- `toggleEmptyContainer(subkitId)`:
  - If already in emptyContainers: removes it
  - If not: adds to emptyContainers AND clears all itemSelections for that subkitId

### PRD FR9
[Source: docs/prd/2-requirements.md]
- Users shall have the option to order a completely empty subkit container for any selected subkit, including the Custom subkit

### UX Spec Flow 3
[Source: UX Spec Section 3, Flow 3]
- Empty Container → item grid dims (opacity-35, pointer-events none); confirmation inline
