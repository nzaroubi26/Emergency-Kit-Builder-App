# Story 16.1: KitSummaryCard — Custom Path Implementation

## Status: Draft

## Story

**As a** developer,
**I want** the `KitSummaryCard` component's custom path fully implemented,
**so that** users who built their own kit see an accurate summary on the Review & Order page.

## Acceptance Criteria

1. The `KitSummaryCard` `path === 'custom'` branch is implemented, replacing the Sprint 1 scaffold placeholder (`<div>Custom kit summary — Sprint 2</div>` at line 72 of `KitSummaryCard.tsx`).
2. The custom variant reads `selectedSubkits` and `itemSelections` from `useKitStore()`.
3. For each selected subkit, the card displays: category color dot (8px circle, `colorBase`), category name, size (Regular/Large), configured item count, and subkit pricing subtotal (via existing `calculateSubkitCartTotal` from `src/utils/cartCalculations.ts`).
4. Slot count is computed from the user's selections via existing `calculateTotalSlots` from `src/utils/slotCalculations.ts`.
5. Path label displays "Custom Kit · N slots used" (vs. "Essentials Kit · 5 slots used"). Same `text-caption` (12px/400), `neutral-400` styling.
6. Visual structure: two-line rows per subkit. Line 1: color dot + name + size badge. Line 2: item count + " · " + subtotal in `text-caption`, `neutral-500`. Rows separated by `divide-y neutral-200`. Empty subkit: "No items configured" in `neutral-400`.
7. Component tests cover: renders all user-selected subkits, displays correct item counts and subtotals, computes correct slot count, handles empty container edge case, handles subkit with zero items gracefully.

## Dev Notes

### Architecture Reference
[Source: docs/architecture-phase3-sprint2.md, Section 4]

**Data flow:** `useKitStore().selectedSubkits` provides category IDs, sizes, selection order. `useKitStore().itemSelections` provides per-item quantities and inclusion state.

**Per-subkit item count:** Derived inline:
```ts
const itemCount = Object.values(itemSelections)
  .filter((sel) => sel.subkitId === subkit.subkitId && sel.included)
  .length;
```

**Per-subkit subtotal:** Uses existing `calculateSubkitCartTotal(subkit, itemSelections, ITEMS)` from `src/utils/cartCalculations.ts`.

**Slot count:** Uses existing `calculateTotalSlots(selectedSubkits)` from `src/utils/slotCalculations.ts`.

No new utility functions needed.
