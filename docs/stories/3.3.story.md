# Story 3.3: Visualizer Integration with Subkit Selection

## Status: Draft

## Story

**As a** user,
**I want** the housing unit visualizer to respond instantly to my subkit selections and size choices,
**so that** I always have a real-time view of how my kit fills the housing unit.

## Acceptance Criteria

1. The visualizer updates immediately upon any subkit selection, deselection, or size change
2. Selected subkits appear in the visualizer in the correct slot position, color, and size
3. Deselected subkits are immediately removed and their slots revert to the + empty state
4. The visualizer and subkit cards remain in sync at all times
5. Slots fill from top to bottom in the order subkits are selected
6. If a Large subkit is switched to Regular, the freed slot is made available immediately

## Tasks / Subtasks

- [ ] Task 1: Verify end-to-end integration (AC: 1, 2, 3)
  - [ ] Select a category → slot fills with correct color and name
  - [ ] Deselect a category → slot clears to empty + state
  - [ ] Change size → visualizer reflects new slot allocation
  - [ ] All updates happen in real time (< 100ms to begin rendering)
- [ ] Task 2: Verify fill order (AC: 5)
  - [ ] First selected subkit fills slot 0 (top)
  - [ ] Second selected fills next available slot
  - [ ] Large subkits fill 2 consecutive slots
  - [ ] Deselecting mid-sequence: remaining subkits shift up, order preserved
- [ ] Task 3: Verify card-visualizer sync (AC: 4)
  - [ ] Card selected state always matches visualizer slot state
  - [ ] No state where card shows selected but slot is empty or vice versa
  - [ ] SizeToggle state matches visualizer large/regular rendering
- [ ] Task 4: Handle deselect mid-sequence (AC: 3, 5)
  - [ ] When a subkit in the middle is deselected, slots below shift up
  - [ ] Selection order preserved for item configuration sequence
  - [ ] Store `deselectSubkit` re-indexes `selectionOrder`
- [ ] Task 5: Handle size change freed slot (AC: 6)
  - [ ] Large → Regular: slot below becomes available
  - [ ] Previously disabled cards re-enable if capacity drops below 6
  - [ ] Visualizer re-renders with updated slot allocation
- [ ] Task 6: ARIA announcements (AC: 1)
  - [ ] Announce subkit selection: `announcePolite("{name} added to slot {n}")`
  - [ ] Announce deselection: `announcePolite("{name} removed")`
  - [ ] Announce size change: `announcePolite("{name} changed to {size}")`

## Dev Notes

### Integration Architecture
This story is primarily an integration verification story. The individual pieces (store, visualizer, cards) are built in Stories 2.1-2.3 and 3.1-3.2. This story ensures they work together seamlessly.

### Slot Fill Order
[Source: docs/architecture/5-state-management.md#slot-calculation-pure-functions]
- `calculateSlotState()` fills top-to-bottom in `selectionOrder`
- `deselectSubkit()` re-indexes `selectionOrder` on remaining subkits

### ARIA Announcements
[Source: docs/architecture/11-frontend-developer-standards.md#aria-live-region-setup]
- Use `announcePolite()` from `src/utils/announce.ts`
- Call from store action side-effects or component event handlers
