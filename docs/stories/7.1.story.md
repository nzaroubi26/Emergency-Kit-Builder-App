# Story 7.1: "Fill My Kit for Me" Auto-Select

## Status: Done

## Story

**As a** user,
**I want** a single checkbox that auto-selects all items in my current subkit,
**so that** I can quickly populate a sensible default kit without toggling each item individually.

## Acceptance Criteria

1. A "Fill my kit based on expert advice" checkbox is added to both `ItemConfigScreen` and `CustomSubkitScreen`, displayed side by side with the "Send an empty container" checkbox in a 2-column grid layout above the item list.
2. Checking the checkbox calls the existing `toggleItem` store action for every unselected item in the current subkit, setting each to quantity 1.
3. Unchecking the checkbox clears all item selections for the current subkit by calling `toggleItem` for each selected item, returning the subkit to the default empty state.
4. The checkbox is visually consistent with the existing `EmptyContainerOption` component pattern.
5. The checkbox is disabled and visually grayed out when the subkit is set to empty container â€” a subkit cannot be both empty and filled.
6. The Summary Page correctly reflects all auto-selected items and their quantities.

## Dev Notes

### Fill Toggle Logic
- `isAllFilled`: `useMemo` checks if every item in the current subkit has a selection with quantity >= 1
- `handleFillToggle`: When checked, iterates all items and calls `toggleItem` for each unselected item (quantity 1). When unchecked, calls `toggleItem` for each selected item to clear selections.
- Uses existing `toggleItem` store action â€” no new store actions required

### 2-Column Grid Layout
- "Fill my kit based on expert advice" and "Send an empty container" displayed side by side using Tailwind grid utilities
- Visually consistent with existing `EmptyContainerOption` component pattern

### Empty Container Interaction
- When empty container is checked, fill checkbox is disabled and visually grayed out
- Toggling empty container on clears all items regardless of fill state

### Data References
- `src/data/kitItems.ts` provides `ITEMS_BY_CATEGORY` and `STANDARD_CATEGORY_IDS` for iterating items

## Integration Verification
