# Story 12.5: Subkit Subtotal on Summary/Review Kit Page

## Status: Ready for Review

## Story

**As a** kit builder user,
**I want** to see a subtotal for each subkit at the bottom of its summary section on the Review Kit page,
**so that** I can review the cost of each subkit (container plus items) before proceeding.

## Acceptance Criteria

1. **AC1 — Subtotal line displayed at the bottom of each SubkitSummarySection.**
   A subtotal line is rendered at the bottom of the content area in `src/components/summary/SubkitSummarySection.tsx`, below the item list (or below the "Empty Container" badge, or below "No items configured"). Format: `Subtotal: $XX.XX` aligned right.

2. **AC2 — Subtotal typography and color match spec.**
   The subtotal text uses `text-xs` (12px) with `font-normal` (400 weight) and color `text-[var(--color-neutral-400)]` — matching `text-caption` / `neutral-400` from the epic spec.

3. **AC3 — Subtotal computed using `calculateSubkitCartTotal`.**
   The subtotal value is computed using the existing `calculateSubkitCartTotal` pure function from `src/utils/cartCalculations.ts` — not reimplemented inline. Parameters: `subkit` (already available via props), `itemSelections` (needs to be passed or accessed), `allItems` (needs to be passed or accessed).

   The subtotal is computed by each parent screen (`SummaryScreen.tsx` and `OrderConfirmationScreen.tsx`) using `calculateSubkitCartTotal(subkit, itemSelections, ITEMS)` and passed to `SubkitSummarySection` as a `subtotal` prop. `SubkitSummarySection` remains a presentational component with no Zustand store dependency — it does not import `calculateSubkitCartTotal` itself.

   Both screens render `SubkitSummarySection` with the same data. The subtotal prop must be wired in both `SummaryScreen.tsx` and `OrderConfirmationScreen.tsx`.

### calculateSubkitCartTotal Function
[Source: src/utils/cartCalculations.ts]

```typescript
export function calculateSubkitCartTotal(
  subkit: SubkitSelection,
  itemSelections: Record<string, ItemSelection>,
  allItems: KitItem[]
): number {
  let total = CONTAINER_PRICES[subkit.size];
  for (const sel of Object.values(itemSelections)) {
    if (sel.subkitId === subkit.subkitId && sel.included) {
      const matchingItem = allItems.find((i) => i.id === sel.itemId);
      if (matchingItem) {
        total += calculateItemLineTotal(matchingItem.pricePlaceholder, sel.quantity);
      }
    }
  }
  return total;
}
```

### Design Decision: Pre-computed Required Prop

`SubkitSummarySection` is a presentational component — it has no Zustand store dependency. To keep it presentational, the subtotal is pre-computed by each parent screen and passed as a required prop (`subtotal: number`). This is consistent with the existing pattern where `weightLbs` and `volumePct` are also computed by the parent and passed as required props. Making it required (not optional) ensures TypeScript will catch any callsite that forgets to pass it.
