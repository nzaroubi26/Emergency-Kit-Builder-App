# Story 2.1: Static Visualizer Component

## Status: Draft

## Story

**As a** user,
**I want** to see a visual representation of my housing unit with 6 empty slots,
**so that** I can understand the physical structure of the storage system before making any selections.

## Acceptance Criteria

1. A self-contained Visualizer React component is created and rendered at the top of the Subkit Selection screen
2. The visualizer displays a single vertical column of 6 equal-sized rectangular slots, stacked top to bottom
3. Each empty slot displays a + icon centered within it
4. The visualizer is styled to resemble the interior of a rectangular storage container
5. The component accepts slot state as props — fully controlled and stateless internally
6. The visualizer renders correctly on desktop and tablet screen sizes
7. Component is documented with a clear props interface in TypeScript

## Dev Notes

### Component Architecture
[Source: docs/architecture/11-frontend-developer-standards.md, Rule 10]
- HousingUnitVisualizer receives NO store references
- Takes only `slots: SlotState[]`, `readOnly?`, and `onSlotClick?`
- All derivation happens in the parent component
- This is critical for NFR6 — self-contained extensible module

### Props Interface
[Source: docs/architecture/5-state-management.md#type-definitions]
```typescript
interface HousingUnitVisualizerProps {
  slots: SlotState[];              // Always length 6; index 0 = top slot
  readOnly?: boolean;              // true on Summary Page
  onSlotClick?: (slotIndex: number) => void;  // Phase 2 — wired, dormant in MVP
}
```

### SlotState Type
[Source: docs/architecture/5-state-management.md#type-definitions]
```typescript
interface SlotState {
  status: 'empty' | 'filled';
  subkitId?: string;
  subkitName?: string;
  subkitColor?: string;    // Hex from category colorBase
  size: SubkitSize;
  isLargeStart?: boolean;
  isLargeEnd?: boolean;
}
```

### Slot Fill Direction
[Source: docs/prd/6-epic-details.md, UX Spec Decision #2]
- Top-to-bottom fill order (NOT bottom-up — this is a brownfield correction)

### Visualizer Placement
[Source: UX Spec Decision #9]
- Stacked above cards, centered, max-w-sm

### Animation (for later stories but good context)
[Source: docs/architecture/8-styling-guidelines.md#animation-rules]
- Slot fill: `background-color` transition over 220ms
- Name label: `opacity` transition delayed 80ms
- GPU-composited properties only (transform, opacity)

### Testing

- Full test template provided in architecture for this component
- Tests: renders 6 slots, shows subkit name when filled, hides + in readOnly, 44px height in readOnly
- Must include axe accessibility assertion
