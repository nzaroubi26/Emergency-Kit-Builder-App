# Story 2.4: Visualizer Extensibility for Phase 2

## Status: Draft

## Story

**As a** developer,
**I want** the visualizer component architected for future clickable slot interactivity,
**so that** Phase 2 slot-click-to-assign functionality can be added without rebuilding the component.

## Acceptance Criteria

1. The visualizer exposes an optional onSlotClick callback prop — dormant in MVP but wired for Phase 2
2. Each slot is rendered as an individually addressable element with a unique slot index
3. Component architecture is documented noting Phase 2 extension points
4. No regressions introduced to existing slot rendering or state management
5. Code is reviewed and approved as Phase 2 ready by the development team

## Tasks / Subtasks

- [ ] Task 1: Verify onSlotClick prop wiring (AC: 1)
  - [ ] Confirm `HousingUnitVisualizer` accepts `onSlotClick?: (slotIndex: number) => void`
  - [ ] Confirm `VisualizerSlot` receives and can call `onSlotClick` with its `slotIndex`
  - [ ] In MVP: no parent passes `onSlotClick` — prop is dormant
  - [ ] When `onSlotClick` is provided, slots should have `cursor-pointer` and `role="button"`
- [ ] Task 2: Verify slot addressability (AC: 2)
  - [ ] Each slot has `data-testid="slot-{index}"` where index is 0-5
  - [ ] Each slot has a unique key based on slot index
  - [ ] Slots are individually queryable in tests
- [ ] Task 3: Document Phase 2 extension points (AC: 3)
  - [ ] Add JSDoc comments on `HousingUnitVisualizerProps` describing Phase 2 behavior of `onSlotClick`
  - [ ] Document in component file: how to enable clickable slots, expected handler behavior
  - [ ] Note: Phase 2 will add click-to-assign where clicking empty slot triggers subkit assignment flow
- [ ] Task 4: Regression testing (AC: 4)
  - [ ] Run all existing visualizer tests — zero failures
  - [ ] Run all slot calculation tests — zero failures
  - [ ] Verify filled, empty, large, mixed states all render correctly
  - [ ] Verify readOnly mode still works (no click handlers, no cursors)
- [ ] Task 5: Write extensibility tests (AC: 1, 2)
  - [ ] Test: onSlotClick is called with correct slotIndex when slot is clicked
  - [ ] Test: onSlotClick is not called when not provided
  - [ ] Test: onSlotClick is not called in readOnly mode

## Dev Notes

### NFR6 Compliance
[Source: docs/prd/2-requirements.md]
- The visualizer component shall be built as a self-contained, extensible module
- Phase 2 adds clickable slot interactivity without full rebuild

### Architecture Rule 10
[Source: docs/architecture/11-frontend-developer-standards.md, Rule 10]
- HousingUnitVisualizer receives no store references
- Takes only `slots: SlotState[]`, `readOnly?`, and `onSlotClick?`
- All derivation happens in the parent — this ensures the component can be reused in any context

### Phase 2 Extension Pattern
The onSlotClick callback enables Phase 2 to add:
- Click empty slot → open subkit assignment picker
- Click filled slot → open edit/remove options
- All without modifying the visualizer component itself

### Testing

[Source: docs/architecture/9-testing-requirements.md]
- Regression: all existing tests must pass
- New tests for onSlotClick behavior
- axe accessibility assertion required
