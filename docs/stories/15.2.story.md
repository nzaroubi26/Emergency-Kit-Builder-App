# Story 15.2: Visualizer Layout Refresh ‚Äî Vertical to Horizontal

## Status: Ready for Review

## Story

**As a** user,
**I want** the kit builder screen to display the housing unit and subkit options side-by-side,
**so that** I can see my selections and available options simultaneously without scrolling.

## Acceptance Criteria

1. SubkitSelectionScreen layout is updated: housing unit visualizer on the left (40%), subkit card list on the right (60%) on desktop (‚â•1024px). Stacked on tablet/mobile (visualizer top, cards below).
2. The housing unit visualizer component (`HousingUnitVisualizer`) is unchanged internally. Its `max-w-sm` (384px) fits within the 40% column at 1024px. No interface changes needed.
3. SubkitCards are displayed in a single-column scrollable list within the right panel (desktop) or a 2-column grid (tablet/mobile), per Sally's Sprint 2 spec.
4. Slot usage indicator (`SlotUsageBar`), `SlotFullIndicator` warning, and "Configure Items" CTA are pinned in the left column below the visualizer (desktop). On stacked mobile, they sit between the visualizer and the card list.
5. All existing functionality is preserved: select/deselect, size toggle, slot count impact, capacity enforcement, "Configure Items" navigation.
6. Category color updates (if any) are applied to `CATEGORIES` `colorBase`/`colorTint` values per Sally's Sprint 2 spec. *(Sally Decision #5: no existing color changes ‚Äî Sprint 2 color work is additive only via Pets.)*
7. Left column is sticky on desktop (`position: sticky; top: 1.5rem`) so the visualizer stays visible while scrolling the card list.
8. Component tests updated for new layout assertions. All existing behavioral tests still pass.
