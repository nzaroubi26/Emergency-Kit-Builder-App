# Story 4.5: Navigation and Flow Completion

## Status: Draft

## Story

**As a** user,
**I want** to move smoothly through all my subkit configurations and proceed to my summary when done,
**so that** I can complete my kit configuration efficiently and without confusion.

## Acceptance Criteria

1. A clearly labeled Next Subkit CTA advances the user to the next subkit in their sequence
2. On the final subkit, the CTA changes to Review My Kit — transitioning to the Summary Page
3. A Back button allows the user to return to the previous subkit without losing selections
4. A Back to Subkit Selection option is available with a confirmation prompt
5. All item selections, quantities, and empty container choices are preserved throughout the flow
6. The progress indicator updates correctly at every step

## Dev Notes

### Navigation Flow
[Source: docs/architecture/7-routing.md#navigation-flow]
```
/configure/:subkitId → [Next Subkit] → /configure/:nextSubkitId
/configure/:subkitId → [Next Subkit — if Custom] → /configure/custom
/configure/:lastSubkitId → [Review My Kit] → /summary
```

### Config Sequence
[Source: docs/architecture/5-state-management.md#store]
- `currentConfigIndex` tracks position (0-indexed)
- Selected subkits ordered by `selectionOrder`
- Navigate by resolving `selectedSubkits[currentConfigIndex + 1].subkitId`

### Confirmation Modals
[Source: UX Spec Section 2.2]
- Back to Subkit Selection: "Going back will keep all your selections. You can return at any time."
- Focus-trapped dialog; Escape = cancel only

### Screen Transitions
[Source: docs/architecture/7-routing.md#screen-transition-directions]
- Forward: exit translateX(-16px) + fade; enter from +16px
- Back: reverse direction
- Implemented at router level

### Testing

[Source: docs/architecture/9-testing-requirements.md]
- Test CTA label changes based on position
- Test navigation between subkits
- Test state preservation across navigation
- Manual E2E for full flow
