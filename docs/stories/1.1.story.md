# Story 1.1: Project Scaffolding and Setup

## Status: Approved

## Story

**As a** developer,
**I want** a fully configured React project with Tailwind CSS, routing, and a deployment pipeline,
**so that** the team has a clean, working foundation to build all features upon.

## Acceptance Criteria

1. React application initialized with TypeScript support
2. Tailwind CSS configured and working with a sample utility class rendered on screen
3. React Router configured with placeholder routes for Kit Builder, Item Configuration, and Summary Page
4. Project builds to static output in `dist/` and is deployable to the hosting platform (Replit or Vercel)
5. Basic README documents how to run, build, and deploy the project locally

## Dev Notes

### Tech Stack
[Source: docs/architecture/2-frontend-tech-stack.md]
- **Build tool:** Vite 6.x
- **Framework:** React 18.x + TypeScript 5.x strict
- **Styling:** Tailwind CSS v4.x with CSS custom properties
- **State management:** Zustand 5.x (installed now, configured in Story 1.2)
- **Routing:** React Router 6.4+ Data Router API
- **Icons:** lucide-react (named imports only)
- **Testing:** Vitest 2.x + React Testing Library 16.x + @axe-core/react 4.x
- **Linting:** ESLint 8.x + @typescript-eslint + eslint-plugin-jsx-a11y 6.x
- **Formatting:** Prettier 3.x
- **Deployment:** Static SPA hosting (Replit or Vercel — builds to `dist/`)

### Project Structure
[Source: docs/architecture/3-project-structure.md]
Full directory tree defined — see Task 5 for skeleton. Key entry points:
- `src/main.tsx` — React.StrictMode + RouterProvider
- `src/App.tsx` — RouterProvider root
- `index.html` — Vite entry

### Routing Configuration
[Source: docs/architecture/7-routing.md]
- Use `createBrowserRouter` from `react-router-dom`
- Routes: `/` (SubkitSelectionScreen), `/configure/:subkitId` (ItemConfigScreen), `/configure/custom` (CustomSubkitScreen), `/summary` (SummaryScreen), `*` catch-all redirect to `/`
- Guard functions in `src/router/guards.ts` — placeholder stubs for now (guards fully implemented in later stories)
- AppShell wraps all routes via `<Outlet />`

### Styling
[Source: docs/architecture/8-styling-guidelines.md]
- `globals.css` must contain `@import "tailwindcss"` and a `@theme` block with all design tokens (brand colors, neutrals, semantic colors, radii, shadows, motion durations, easings)
- `:focus-visible` and `prefers-reduced-motion` rules must be in globals.css
- Import Inter font from Google Fonts
- Dynamic category colors use inline styles, not Tailwind template literals

### Component Standards
[Source: docs/architecture/4-component-standards.md]
- Named exports only — no `export default`
- `FC<Props>` typed components
- Props interface `{ComponentName}Props` co-located in same file
- No logic in JSX — extract to `const` above return
- PascalCase `.tsx` files, camelCase `.ts` for utils/hooks

### Environment
[Source: docs/architecture/10-environment-configuration.md]
- Static SPA — no backend, no secrets in MVP
- Single env var: `VITE_PURCHASE_URL` (placeholder)
- Typed access via `src/tokens/env.ts` — never use `import.meta.env` directly in components

### Brownfield Context
[Source: docs/architecture/1-template-and-framework-selection.md]
- This is a brownfield refactor. The existing project is a Next.js application that must be replaced with a Vite + React scaffold.
- Task 0 removes all Next.js-specific files and dependencies. Non-source directories (`docs/`, `.bmad-core/`, `attached_assets/`) are preserved.
- Story 1.1 establishes the target scaffold. Existing brownfield files (`SubkitTypeSelectionNew.tsx`, `SummaryPage.tsx`, `ItemQuestionnaireFlow.tsx`, `kitItems.ts`, `ImageWithFallback`) are corrected in later stories where logically required — not in this story.
- Pre-refactor tag (`v0-pre-refactor`) should be created before any story begins.

### Tailwind v4 + Vite Integration
- Tailwind CSS v4 uses CSS-first configuration — no `tailwind.config.ts` file needed.
- Install `tailwindcss` and `@tailwindcss/vite` (not `postcss` or `autoprefixer`).
- Add `@tailwindcss/vite` plugin to `vite.config.ts` alongside `@vitejs/plugin-react`.
- All design tokens are defined in the `@theme` block inside `src/styles/globals.css`.
- Inter font is imported via `@import url(...)` at the top of `globals.css`.

### Testing

[Source: docs/architecture/9-testing-requirements.md]
- Test location: `tests/unit/` for unit tests, `tests/components/` for component tests
- Framework: Vitest 2.x + React Testing Library 16.x
- `vitest.config.ts` uses jsdom environment, globals: true, setup file at `tests/setup.ts`
- `tests/setup.ts` imports `@testing-library/jest-dom` and configures `testIdAttribute: 'data-testid'`
- No snapshot tests allowed
- Every component test file must include at least one axe accessibility assertion
