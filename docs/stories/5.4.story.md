# Story 5.4: Summary Page Presentation and Print Readiness

## Status: Draft

## Story

**As a** user,
**I want** my kit summary to be clearly formatted and easy to save or reference later,
**so that** I can keep a record of my configuration for future reference.

## Acceptance Criteria

1. The Summary Page layout is clean and well-structured for easy reading on desktop and tablet
2. The page is print-friendly — a browser print produces a clean, readable document
3. Section headings, subkit names, and item lists are clearly hierarchically structured
4. The page does not include unnecessary UI chrome that would distract from the summary content
5. A Start Over option is available with a confirmation prompt to reset the entire configuration

## Tasks / Subtasks

- [ ] Task 1: Polish Summary Page layout (AC: 1, 3)
  - [ ] Clean section hierarchy: kit overview → visualizer → subkit sections
  - [ ] Consistent spacing using Tailwind utilities
  - [ ] Readable typography with clear heading levels
  - [ ] Responsive layout: desktop and tablet
- [ ] Task 2: Add Print button (AC: 2, 4)
  - [ ] SecondaryButton: "Print My Kit"
  - [ ] On click: `window.print()`
  - [ ] No separate route — uses CSS @media print
  - [ ] CSS class `.btn-print` for self-hiding in print
- [ ] Task 3: Verify print output (AC: 2, 4)
  - [ ] Hidden in print: header, step indicator, all CTA/action buttons
  - [ ] Visible in print: kit overview, visualizer, subkit sections with items
  - [ ] `page-break-inside: avoid` on subkit sections
  - [ ] Clean white background, 12pt font, black text
  - [ ] Empty visualizer slots render as clean boxes (no + icons)
- [ ] Task 4: Add Start Over option (AC: 5)
  - [ ] SecondaryButton: "Start Over"
  - [ ] On click: show ConfirmationModal
  - [ ] Modal message: "Starting over will clear your entire kit configuration. Are you sure?"
  - [ ] Confirm: call `resetKit()` from store → navigate to `/`
  - [ ] Cancel: dismiss modal, stay on Summary Page
  - [ ] CSS class `.btn-start-over` for print hiding
- [ ] Task 5: Final layout review (AC: 1, 3, 4)
  - [ ] No unnecessary UI chrome in summary view
  - [ ] Subkit sections visually distinct with category color accents
  - [ ] Overall professional, organized appearance

### Print CSS Implementation
[Source: docs/architecture/8-styling-guidelines.md#print-css]
```css
@media print {
  header, .step-progress-indicator,
  .btn-get-my-kit, .btn-edit, .btn-print, .btn-start-over { display: none !important; }
  body { background: white; font-size: 12pt; color: #000; }
  .summary-subkit-section { page-break-inside: avoid; }
  .visualizer-slot-empty::after { content: none; }
  .visualizer-slot-empty { background-color: #f8f9fa !important; border: 1px solid #e5e7eb !important; }
}
```
