# Story 14.3: Essentials Bundle Config, Route Guards, MobileInterstitial Bypass + Full Flow Wiring

## Status: Draft

## Story

**As a** developer,
**I want** the Essentials bundle defined as a typed config constant, route guards enforcing sequential flow, MobileInterstitial bypassed for Phase 3 routes, and the full Sprint 1 flow wired end-to-end,
**so that** the bundle is easy to update, flow integrity is enforced, and mobile users can access Phase 3 screens.

### Route Guards
[Source: docs/architecture-phase3/4-route-guards.md]

```typescript
import { useMCQStore } from '../store/mcqStore';

export function mcqHouseholdGuard() {
  const { emergencyTypes } = useMCQStore.getState();
  if (emergencyTypes.length === 0) return redirect('/build');
  return null;
}

export function forkGuard() {
  const { emergencyTypes, householdComposition } = useMCQStore.getState();
  if (emergencyTypes.length === 0 || householdComposition.length === 0) {
    return redirect('/build');
  }
  return null;
}

export function reviewGuard() {
  const { kitPath } = useMCQStore.getState();
  if (!kitPath) return redirect('/choose');
  return null;
}
```

### MobileInterstitial Bypass
[Source: docs/architecture-phase3/5-mobileinterstitial-exemption.md]

Current implementation in `AppShell.tsx`:
```typescript
{isMobile ? <MobileInterstitial /> : <Outlet />}
```

Updated to:
```typescript
const MOBILE_EXEMPT_ROUTES = ['/build', '/build/household', '/choose', '/review'];

const isMobileExempt = MOBILE_EXEMPT_ROUTES.some(
  (route) => location.pathname === route || location.pathname.startsWith(route + '/')
);

// In JSX:
{isMobile && !isMobileExempt ? <MobileInterstitial /> : <Outlet />}
```
