# Story 12.4: Subkit Subtotal on Item Configuration Screen

## Status: Ready for Review

## Story

**As a** kit builder user,
**I want** to see a live subtotal for the current subkit at the bottom of the Item Configuration screen,
**so that** I have continuous cost visibility as I add, remove, or adjust items without needing to open the cart sidebar.

## Acceptance Criteria

1. **AC1 — Subtotal indicator displayed on ItemConfigScreen.**
   A subtotal indicator is rendered at the bottom-right of `src/components/item-config/ItemConfigScreen.tsx`, below the item card grid and above the navigation buttons. Format: `Subkit Subtotal: $XX.XX` where the dollar amount is `calculateSubkitCartTotal(currentSubkit, itemSelections, allItems).toFixed(2)`.

2. **AC2 — Subtotal indicator displayed on CustomSubkitScreen.**
   The same subtotal indicator is rendered at the bottom-right of `src/components/item-config/CustomSubkitScreen.tsx` in the equivalent position. It uses the same `calculateSubkitCartTotal` call with the custom subkit's data.

3. **AC3 — Subtotal typography and color match spec.**
   The subtotal text uses `text-xs` (12px) with `font-normal` (400 weight) and color `text-[var(--color-neutral-500)]` — matching `text-caption` / `neutral-500` from the epic spec.

4. **AC4 — Subtotal updates live as selections change.**
   The subtotal recalculates reactively when items are added/removed (via `toggleItem`), quantities change (via `setItemQuantity`), or the empty container toggle changes. No manual refresh or navigation is required.

5. **AC5 — Minimum value equals container price.**
   When no items are included (or the empty container option is checked), the subtotal displays the base container price for the subkit's selected size: `$40.00` for Regular, `$60.00` for Large. This is the natural result of `calculateSubkitCartTotal` which always starts with `CONTAINER_PRICES[subkit.size]`.

6. **AC6 — `calculateSubkitCartTotal` imported from `cartCalculations.ts`.**
   The subtotal is computed using the existing `calculateSubkitCartTotal` pure function from `src/utils/cartCalculations.ts` — not reimplemented inline.

7. **AC7 — No changes to component props interfaces.**
   Both `ItemConfigScreen` and `CustomSubkitScreen` already have access to `selectedSubkits`, `itemSelections`, and items data via existing store subscriptions and data imports. No new props are required.

8. **AC8 — No changes to Zustand store shape or `cartCalculations.ts`.**
   The store shape is unchanged. `cartCalculations.ts` is consumed read-only — no modifications.

9. **AC9 — Subtotal indicator is always visible once the subkit is active.**
   The indicator renders regardless of whether items are included, the empty container is selected, or no selections have been made. It is never hidden or conditionally removed.

10. **AC10 — New tests added for subtotal indicator.**

    New tests added to appropriate test locations:

    | Test | Assertion |

    | Subtotal updates when item is toggled | After toggling an item with `pricePlaceholder: 29.99`, subtotal text reflects `$69.99` (container $40 + item $29.99) |
    | Subtotal renders on CustomSubkitScreen | Text matching `Subkit Subtotal:` is present |
    | Subtotal shows Large container price | For a Large subkit with no items, subtotal shows `$60.00` |

### calculateSubkitCartTotal Function
[Source: src/utils/cartCalculations.ts]

```typescript
export function calculateSubkitCartTotal(
  subkit: SubkitSelection,
  itemSelections: Record<string, ItemSelection>,
  allItems: KitItem[]
): number {
  let total = CONTAINER_PRICES[subkit.size];

  for (const sel of Object.values(itemSelections)) {
    if (sel.subkitId === subkit.subkitId && sel.included) {
      const matchingItem = allItems.find((i) => i.id === sel.itemId);
      if (matchingItem) {
        total += calculateItemLineTotal(matchingItem.pricePlaceholder, sel.quantity);
      }
    }
  }

  return total;
}
```

Key behaviors:
- Always starts with `CONTAINER_PRICES[subkit.size]` ($40 Regular, $60 Large) — this is the minimum value.
- Iterates all `itemSelections`, filtering by `subkitId` and `included === true`.
- Uses `calculateItemLineTotal` for each matching item.
- Parameters: `subkit` (SubkitSelection object), `itemSelections` (full store record), `allItems` (KitItem array).

### CONTAINER_PRICES Constant
[Source: src/utils/cartCalculations.ts]

```typescript
export const CONTAINER_PRICES: Record<'regular' | 'large', number> = {
  regular: 40,
  large: 60,
};
```

The local `allItems` is used for rendering category groups, while `calculateSubkitCartTotal` needs the full global `ITEMS` array to match `itemId` lookups correctly.
