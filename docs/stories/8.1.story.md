# Story 8.1: Star Rating System

## Status: Done

## Story

**As a** user,
**I want** to see star ratings and review counts for each item I can select,
**so that** I can make more confident choices about what to include in my kit.

## Acceptance Criteria

1. KitItem type extended with rating: number | null and reviewCount: number | null.
2. All 28 items populated with hardcoded ratings (3.8–4.9) and reviewCounts (89–731).
3. StarRating component at src/components/ui/StarRating.tsx renders filled/partial/empty stars, numeric rating, review count.
4. Uses --color-brand-accent for filled stars and --color-neutral-300 for empty stars.
5. aria-label conveying full rating for screen readers.
6. ItemCard renders StarRating below item name/description when rating is non-null.
7. Rendered on both ItemConfigScreen and CustomSubkitScreen.
8. NOT rendered in SubkitSummarySection on Summary Page.
9. StarRating component test covering star fill, aria-label, and axe accessibility.

## Dev Notes

### KitItem Type Extension
[Source: docs/prd-phase2.md CR4]
- Two new nullable fields added to KitItem: `rating: number | null` and `reviewCount: number | null`
- All existing consumers of KitItem are unaffected — the fields are additive

### StarRating Component
[Source: docs/prd-phase2.md FR9, FR10, FR11]
- Percentage-fill approach for partial stars (e.g., 4.3 fills 60% of the 5th star)
- Uses `role="img"` with descriptive `aria-label` (e.g., "Rated 4.3 out of 5 based on 128 reviews")
- Design tokens: `--color-brand-accent` (#22C55E) for filled, `--color-neutral-300` for empty

### Data Source
[Source: docs/prd-phase2.md Section 4]
- Phase 2 uses hardcoded static values — proof of concept ahead of Bazaarvoice integration in Phase 3
- StarRating and ItemCard integration are data-source agnostic — swapping to live data requires only changes to the data layer

### Summary Page Exclusion
[Source: docs/prd-phase2.md FR11]
- SubkitSummarySection does not render StarRating — star ratings are a selection aid, not a summary element

### Testing
[Source: docs/architecture/9-testing-requirements.md]
- 8 tests in StarRating.test.tsx covering rendering, accessibility, and star fill
- Existing ItemCard.test.tsx tests verify rating display integration
- axe assertion required for accessibility compliance
