# Story 17.2: Static Product Data File and Affiliate Link Utility

## Status: Review

## Story

**As a** developer,
**I want** a typed static data file mapping all 31 Amazon products to their subkit categories with ASINs, prices, and images, and a utility to construct affiliate links,
**so that** the Fill Your Kit screen has a complete data source and can generate working Amazon links.

## Acceptance Criteria

1. A new file `src/data/amazonProducts.ts` exports a typed array and pre-indexed lookup:
   ```ts
   export interface AmazonProduct {
     id: string;              // matches kitItems.ts item ID
     categoryId: string;      // matches CATEGORIES key
     kitItemId: string;       // matches ITEMS id (same as id for 1:1 mapping)
     name: string;            // generic product name
     asin: string;            // Amazon Standard Identification Number (10 chars)
     price: number;           // point-in-time snapshot
     brand: string;           // brand/product line
     imageSrc: string;        // '/products/{ASIN}.jpg'
     mcqCondition?: {
       field: 'householdComposition';
       includes: HouseholdOption;
     };
   }
   export const AMAZON_PRODUCTS: AmazonProduct[];
   export const PRODUCTS_BY_CATEGORY: Record<string, AmazonProduct[]>;
   ```
2. All 31 products from `docs/sprint-3-product-catalog.md` are defined with correct ASINs and prices.
3. Kids Rain Poncho has `mcqCondition: { field: 'householdComposition', includes: 'kids' }`.
4. All Pets category items have `mcqCondition: { field: 'householdComposition', includes: 'pets' }`.
5. Product images are downloaded and placed in `public/products/`, named by ASIN (e.g., `B082TMBYR6.jpg`). Referenced in `amazonProducts.ts` as `/products/B082TMBYR6.jpg`. Image download is a dev task — download each product's primary image from its Amazon listing page and save as `{ASIN}.jpg`.
6. A new file `src/utils/affiliateLink.ts` exports:
   ```ts
   export const AFFILIATE_TAG = 'placeholder-20';
   export function buildAffiliateUrl(asin: string, tag?: string): string
   ```
   Returns `https://www.amazon.com/dp/${asin}?tag=${tag || AFFILIATE_TAG}`.
7. Unit tests cover: `buildAffiliateUrl` returns correct URL format, handles custom tag override, all 31 products have valid ASINs (non-empty, 10 chars), all products have `price > 0`, all products reference a valid `categoryId`.

## Dev Notes

### Architecture Reference
[Source: docs/architecture-phase3-sprint3.md, Section 1]

The `AmazonProduct` interface and data shape are fully specified in the architecture brief. Key design decisions:
1. `id === kitItemId` for all 31 entries (1:1 mapping in Sprint 3A)
2. `mcqCondition` is declarative — the rendering component checks `householdComposition.includes(product.mcqCondition.includes)`
3. `PRODUCTS_BY_CATEGORY` mirrors `ITEMS_BY_CATEGORY` reduce pattern
4. `imageSrc` is root-relative: `/products/{ASIN}.jpg`
5. Price is a raw number — formatting (`$XX.XX`) is a component concern

### Affiliate Link Utility
[Source: docs/architecture-phase3-sprint3.md, Section 2]

- `AFFILIATE_TAG` is a named constant, not an env var (per PRD CR4)
- `tag` parameter uses `encodeURIComponent` for safety
- `cartUrl.ts` (Story 17.4) imports `AFFILIATE_TAG` from this file — single source of truth

### Image Storage
[Source: docs/architecture-phase3-sprint3.md, Section 6]

Product images go in `public/products/` — these are data-adjacent content assets, not component assets. Referenced by path string, no import resolution needed. Naming convention: `{ASIN}.{ext}`.

### Product Catalog Reference
[Source: docs/sprint-3-product-catalog.md]

All 31 products with ASINs, prices, brands, and category mappings are in the product catalog. Use this as the definitive data source for populating `AMAZON_PRODUCTS`.
