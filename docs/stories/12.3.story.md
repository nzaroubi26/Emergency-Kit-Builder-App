# Story 12.3: Item Prices and Line Totals on SubkitSummarySection

## Status: Ready for Review

## Story

**As a** kit builder user,
**I want** to see per-item prices and line totals alongside each item on the Summary/Review Kit page,
**so that** I can review the cost breakdown of my configured kit before proceeding to checkout.

## Acceptance Criteria

1. **AC1 — Per-item price displayed on each item row.**
   In `src/components/summary/SubkitSummarySection.tsx`, each item row in the item list displays the per-unit price alongside the existing quantity badge. For items with qty 1: display `$XX.XX each`. For items with qty > 1: display `$XX.XX × N = $YY.YY` where `$XX.XX` is the per-unit price and `$YY.YY` is the line total computed via `calculateItemLineTotal(item.pricePlaceholder, quantity)`.

2. **AC2 — Price typography and color match spec.**
   The price/line-total text uses `text-xs` (12px) with `font-normal` (400 weight) and color `text-[var(--color-neutral-400)]` — matching `text-caption` / `neutral-400` from the front-end spec Summary Page element specs.

3. **AC3 — No price information for items with `null` pricePlaceholder.**
   When `item.pricePlaceholder` is `null`, no price text is rendered on that item row. The existing item name and quantity badge continue to display unchanged.

4. **AC4 — `calculateItemLineTotal` imported from `cartCalculations.ts`.**
   The line total is computed using the existing `calculateItemLineTotal` pure function from `src/utils/cartCalculations.ts` — not reimplemented inline.

5. **AC5 — No changes to `SubkitSummarySectionProps` interface.**
   The price data is already available via `items: Array<{ item: KitItem; quantity: number }>` — `item.pricePlaceholder` is accessible without any new props. `calculateItemLineTotal` is a standalone import.

6. **AC6 — Existing item layout structure preserved.**
   The existing item name (`text-sm text-neutral-700`) and quantity badge (colored pill with `×N`) remain unchanged. The price text is added as an additional element in the item row — not replacing any existing content.

7. **AC7 — Prices visible on SummaryScreen and OrderConfirmationScreen.**
   Both screens render `SubkitSummarySection` with the same props. The price display appears on both with no additional wiring.

8. **AC8 — New tests added for SubkitSummarySection price display.**

   New tests added to an appropriate test location:

   | Test | Assertion |
   |---|---|
   | Price renders for item with qty 1 | `screen.getByText('$29.99 each')` present for item with `pricePlaceholder: 29.99` and qty 1 |
   | Line total renders for item with qty > 1 | Text matching `$29.99 × 2 = $59.98` present for item with `pricePlaceholder: 29.99` and qty 2 |
   | No price for item with null pricePlaceholder | No `$` text rendered for item with `null` price |
   | axe accessibility passes with prices displayed | `expect(results).toHaveNoViolations()` |

9. **AC9 — TypeScript strict compilation and all tests pass.**
   `npm run typecheck` and `npm run test:run` both pass with zero errors.

### calculateItemLineTotal Function
[Source: src/utils/cartCalculations.ts]

```typescript
export function calculateItemLineTotal(
  pricePlaceholder: number | null,
  quantity: number
): number {
  if (!pricePlaceholder) {
    return 0;
  }
  return pricePlaceholder * quantity;
}
```

Import this and use it for qty > 1 line total calculations. It returns `price * quantity` for non-null prices, `0` for null.
