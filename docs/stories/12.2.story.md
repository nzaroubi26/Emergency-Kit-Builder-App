# Story 12.2: Container Price Display on EmptyContainerOption

## Status: Approved

## Story

**As a** kit builder user,
**I want** to see the price of an empty container displayed next to the checkbox when I choose the empty container option,
**so that** I know exactly how much the empty container costs based on my selected subkit size before committing.

## Acceptance Criteria

1. **AC1 — Container price displayed inline with checkbox label.**
   In `src/components/item-config/EmptyContainerOption.tsx`, a price element renders below the existing label text "Send an empty container". The price reads `$40.00` for Regular size or `$60.00` for Large size, computed as `'$' + CONTAINER_PRICES[containerSize].toFixed(2)`.

2. **AC2 — Price sourced from `CONTAINER_PRICES` constant.**
   The price value is sourced from the existing `CONTAINER_PRICES` constant imported from `src/utils/cartCalculations.ts`. No hardcoded price values in the component.

3. **AC3 — Price typography and color match spec.**
   The price element uses `text-xs` (12px) with `font-normal` (400 weight) and color `text-[var(--color-neutral-500)]` — matching `text-caption` / `neutral-500` from the front-end spec.

4. **AC4 — New `containerSize` prop added to `EmptyContainerOptionProps`.**
   A new required prop `containerSize: 'regular' | 'large'` is added to `EmptyContainerOptionProps`. This is the only interface change — it is additive and does not break existing props.

5. **AC5 — Parent components pass `containerSize` prop.**
   - `ItemConfigScreen.tsx`: passes `containerSize={currentSubkit?.size ?? 'regular'}` (the `currentSubkit` variable already exists at line 151).
   - `CustomSubkitScreen.tsx`: passes `containerSize={customSubkit?.size ?? 'regular'}` (the `customSubkit` variable already exists at line 153).

6. **AC6 — Price updates reactively if subkit size changes.**
   The displayed price reflects the current subkit size. If the user changes size on a prior screen and returns, the price updates accordingly — this is automatic because the parent reads from the store.

7. **AC7 — Existing EmptyContainerOption tests updated and new tests added.**

   All existing tests in `tests/components/EmptyContainerOption.test.tsx` are updated to pass the new `containerSize` prop. New tests added:

   | Test | Assertion |
   |---|---|
   | Renders $40.00 for regular size | `screen.getByText('$40.00')` present when `containerSize="regular"` |
   | Renders $60.00 for large size | `screen.getByText('$60.00')` present when `containerSize="large"` |
   | axe accessibility passes with price displayed | `expect(results).toHaveNoViolations()` (update existing axe tests to include `containerSize`) |

8. **AC8 — TypeScript strict compilation and all tests pass.**
   `npm run typecheck` and `npm run test:run` both pass with zero errors.
