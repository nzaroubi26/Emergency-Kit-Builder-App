# Story 15.4: MCQ Visual Distinction — Suggested vs. Selected

## Status: Draft

## Story

**As a** user,
**I want** to clearly see which subkits the app is suggesting for my situation versus which ones I've actively chosen,
**so that** I understand the recommendation without feeling pressured.

## Acceptance Criteria

1. SubkitCard receives a new optional `elevated?: boolean` prop. Defaults to falsy — no side effects on existing rendering.
2. When `elevated === true` and the subkit is **not selected**, the card displays:
   - An `ElevationBadge` pill: `Sparkles` icon (10px) + "Suggested for you" text. `brand-primary` (`#1F4D35`) text on `brand-primary-light` (`#E8F5EE`) background, `radius-full`, `px-2 py-0.5`. Positioned below category name, before description.
   - A 3px solid `#22C55E` left border accent replacing the standard left edge.
3. When `elevated === true` and the subkit **is selected**, the elevation badge and left border accent disappear entirely. Standard selected styling takes over (colorTint background, colorBase border, checkmark).
4. When `elevated === false` or undefined, the card renders as it does today — no elevation indicator.
5. The `ElevationBadge` includes the text label "Suggested for you" so the distinction is understandable without color alone (WCAG 2.1 AA).
6. Screen reader: elevated + unselected cards use `aria-label="{categoryName} — Suggested for your situation"`. Elevated + selected cards use `aria-label="{categoryName}, selected"`. Non-elevated: existing behavior.
7. An `ElevationGroupHeader` component renders "Suggested for your situation" (`text-caption`, `neutral-500`) above the elevated group with `mb-3`. A `16px` gap (`mb-4`) separates the last elevated card from the first non-elevated card. Only rendered when elevated subkits exist.
8. Component tests cover: badge renders when elevated + unselected, badge disappears when elevated + selected, badge absent when not elevated, accessibility label present, group header renders when elevated subkits exist, group header absent when none elevated.

## Dev Notes

### Visual Spec
[Source: docs/front-end-spec-phase3-sprint2.md, Section 4]

**Elevated + Unselected:**
| Property | Spec |
|----------|------|
| Left border | 3px solid `#22C55E` |
| Badge pill | `Sparkles` icon (10px) + "Suggested for you", `#1F4D35` on `#E8F5EE`, `radius-full` |
| Card background | `#FFFFFF` (unchanged) |

**Elevated + Selected:**
| Property | Spec |
|----------|------|
| Left border | Standard selected (2px `colorBase`) |
| Badge pill | Hidden |
| Card background | `colorTint` |

**Animation:** Badge fade-out 130ms on selection, fade-in 150ms on deselection. Left border color transitions simultaneously. `prefers-reduced-motion: reduce` collapses to 0.01ms.

### Architecture Reference
[Source: docs/architecture-phase3-sprint2.md, Section 3]

The `elevated` prop is optional, defaults to `undefined` (falsy). All existing rendering paths treat `!elevated` as "standard card". The prop addition is backwards-compatible.
