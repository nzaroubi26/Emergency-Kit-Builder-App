## Dev Notes

### AppShell Architecture
[Source: docs/architecture/3-project-structure.md]
- `AppShell.tsx` is the Outlet wrapper — wraps all child routes
- Contains header + mobile interstitial
- File location: `src/components/layout/AppShell.tsx`

### ARIA Live Regions
[Source: docs/architecture/11-frontend-developer-standards.md#aria-live-region-setup]
- A single `aria-live="polite"` and one `aria-live="assertive"` region mounted in AppShell
- Populated via `src/utils/announce.ts` — not duplicated per component
- Use `requestAnimationFrame` to clear then set textContent for reliable announcement
- Implementation pattern from architecture:
```typescript
let politeEl: HTMLElement | null = null;
let assertiveEl: HTMLElement | null = null;

export function initAnnouncer(polite: HTMLElement, assertive: HTMLElement) {
  politeEl = polite;
  assertiveEl = assertive;
}

export function announcePolite(message: string) {
  if (!politeEl) return;
  politeEl.textContent = '';
  requestAnimationFrame(() => { if (politeEl) politeEl.textContent = message; });
}

export function announceAssertive(message: string) {
  if (!assertiveEl) return;
  assertiveEl.textContent = '';
  requestAnimationFrame(() => { if (assertiveEl) assertiveEl.textContent = message; });
}
```

### Focus Management
[Source: docs/architecture/11-frontend-developer-standards.md#focus-management]
- Every screen component must focus its `<h1>` on mount
- Pattern: `useRef<HTMLHeadingElement>` + `useEffect(() => ref.current?.focus(), [])`
- `<h1>` must have `tabIndex={-1}` and `className="outline-none"`

### Step Progress Indicator
[Source: docs/architecture/3-project-structure.md]
- Shows Step 1 / 2 / 3 — informational, not clickable
- File: `src/components/layout/StepProgressIndicator.tsx`

### MobileInterstitial
- Displayed when viewport is below 768px
- Should inform the user that the kit builder is optimized for desktop/tablet screens
- Not a route — rendered conditionally in AppShell above the Outlet

### Screen Transitions
[Source: docs/architecture/7-routing.md#screen-transition-directions]
- Forward: exit `translateX(-16px)` + fade; enter from `+16px`
- Back: direction reverses
- Duration: `var(--duration-screen)` (240ms)
- Implemented at router level using transition wrapper, not inside individual screens
- NOTE: Screen transitions are NOT in scope for this story — context only for future stories

### Routing
[Source: docs/architecture/7-routing.md]
- Guards are placeholder stubs in this story — full implementation comes in Epic 3+
- Catch-all `*` redirects to `/`
- Router config already exists in `src/router/index.tsx` — no changes needed

### Component Standards
[Source: docs/architecture/4-component-standards.md]
- Named exports only — no `export default`
- `FC<Props>` typed with co-located Props interface
- No logic in JSX — extract to `const` above return

### Testing

[Source: docs/architecture/9-testing-requirements.md]
- Component tests should include axe accessibility assertion
- Test files in `tests/components/`
- Test behavior, not implementation — use `getByRole`, `getByText`
