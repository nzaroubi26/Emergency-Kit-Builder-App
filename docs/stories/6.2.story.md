# Story 6.2: GA4 Analytics Integration

## Status: Done

## Story

**As a** product team,
**I want** key user behavior events tracked automatically,
**so that** we can measure kit completion rates, subkit adoption, item popularity, and CTA conversion.

## Acceptance Criteria

1. An analytics tool (Google Analytics 4) script is added to `AppShell.tsx` as a non-blocking script tag.
2. The analytics tool ID is stored as `VITE_ANALYTICS_ID` in `.env.example` and `src/tokens/env.ts` — never hardcoded.
3. A `src/utils/analytics.ts` module is created exposing a typed `trackEvent` function. Components never call the analytics script directly.
4. Four events instrumented: `kit_completed` (on reaching Summary Page), `subkit_selected` (with `{ categoryId, size }`), `item_included` (with `{ itemId, categoryId }`), and `cta_clicked` (on "Get My Kit" click).
5. All `trackEvent` calls are wrapped in try/catch — analytics failures never surface as user-facing errors.
6. Page load performance is unaffected — analytics script does not block initial render.

## Dev Notes

### Analytics Module Pattern
[Source: docs/architecture/12-phase-2-extension-points.md]
- All analytics calls go through `src/utils/analytics.ts` — components never call GA4 directly
- `injectGA4Script` dynamically injects the GA4 script tag on first mount
- `trackEvent` wraps `gtag('event', ...)` calls with try/catch for resilience

### Environment Variable
[Source: docs/architecture/10-environment-configuration.md]
- `VITE_ANALYTICS_ID` follows Vite convention with `VITE_` prefix
- Accessed via `import.meta.env.VITE_ANALYTICS_ID` in `src/tokens/env.ts`

### Non-blocking Script Injection
[Source: PRD Phase 2 NFR5]
- GA4 script injected via `useEffect` — never blocks initial render
- Script tag uses `async` attribute for non-blocking load

### Event Schema
- `kit_completed`: no properties — fires on Summary Page mount
- `subkit_selected`: `{ categoryId: string, size: string }`
- `item_included`: `{ itemId: string, categoryId: string }`
- `cta_clicked`: no properties — fires on "Get My Kit" click

### Testing
[Source: docs/architecture/9-testing-requirements.md]
- Analytics failures verified via try/catch — never surface to user
- Manual verification: network tab confirms events fire at correct interaction points
