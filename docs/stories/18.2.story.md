# Story 18.2: Fill Your Kit Screen — Product Cards and Affiliate CTAs

## Status: Review

## Story

**As a** user browsing the Fill Your Kit screen,
**I want** to see product cards with images, names, prices, and links to Amazon for each subkit,
**so that** I can purchase the recommended items via affiliate links.

## Acceptance Criteria

1. Within each expanded `CategorySection`, `ProductCard` components are displayed in the responsive grid established in 18.1.
2. Each `ProductCard` displays: product image (square 1:1, 160x160 desktop / 140x140 tablet / 160x160 mobile), generic product name (line-clamp-2), brand as separate muted text (truncate), price formatted as `$XX.XX` (always two decimal places), and a "View on Amazon" CTA button.
3. The "View on Amazon" CTA wraps an `<a>` tag calling `buildAffiliateUrl(product.asin)` with `target="_blank"` and `rel="noopener noreferrer"`.
4. Products with `mcqCondition` are filtered at the screen level (in `FillYourKitScreen`): Kids Rain Poncho hidden if `householdComposition` does not include `'kids'`. Pets items hidden if no pets. `ProductCard` itself does not filter — it receives only visible products.
5. An "Add All to Amazon Cart" CTA button is displayed at the **top of page** (below subtitle, above first category section) **and repeated at the bottom** (below the last category section). Clicking either calls `buildCartUrl()` with the ASINs of all currently displayed products (respecting all filters) and opens the result in a new tab via `window.open(url, '_blank', 'noopener,noreferrer')`. If zero products are displayed (edge case), both CTAs are hidden.
6. A "Prices may vary on Amazon" disclaimer is displayed near each Add All to Cart CTA instance. Style: `text-[12px] text-[var(--color-neutral-400)] text-center`.
7. Product images use `loading="lazy"`. If an image fails to load, display a fallback: `neutral-100` background container with centered `Package` icon (lucide-react, size 48, `neutral-300`).
8. Product cards have hover state: subtle lift (`translateY(-2px)`) + shadow elevation (150ms ease-out). "View on Amazon" CTA uses outlined style (`brand-accent` border and text on white) with fill on hover (`brand-accent` background, white text, 150ms transition).
9. Accessibility:
   - Product links have descriptive `aria-label`: "View {product.name} by {product.brand} on Amazon"
   - Images have `alt={product.name}`
   - "Add All to Amazon Cart" buttons have `aria-label="Add all displayed products to Amazon cart"`
   - All interactive elements keyboard navigable
10. Component tests cover: product card rendering with all fields, affiliate link construction, Add All to Cart URL with correct ASINs, conditional item filtering (kids poncho), image fallback rendering, empty state (category not rendered when all items filtered out).
11. E2E test: complete flow from Order Confirmation → Fill Your Kit → verify at least one product card is visible with a working "View on Amazon" link and "Add All to Amazon Cart" button present at top and bottom of page.

### CTA Contrast Note
[Source: docs/front-end-spec-phase3-sprint3.md, Section 8.6]

The "View on Amazon" button uses `brand-accent` (#22C55E) which does not meet 4.5:1 on white alone (~2.7:1). This is acceptable per WCAG 2.1 SC 1.4.11: the button has a visible 2px border defining its shape, and the label text communicates the action without relying on color.

### 90-Day Cookie Advantage
[Source: docs/sprint-3-api-spike-results.md]

The Add to Cart URL mechanism earns affiliate commission if purchased within 90 days — the "Add All to Amazon Cart" CTA is the highest-value affiliate action on the page. This is why it gets primary button styling and top+bottom placement.
